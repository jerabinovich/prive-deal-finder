OpenClaw — Cutover Checklist

Migración de scripts viejos → scripts consolidados. Seguir en orden exacto.

0 de 24 pasos completados

⚠ Antes de arrancar

Hacé esto en horario de baja actividad (noche o sábado). Todo el cutover toma ~30 minutos. Los scripts viejos se desactivan antes de prender los nuevos — hay una ventana donde Amanda no procesa. Emails no se pierden, quedan en inbox esperando.

Pre-Cutover
0
Backup & Preparación
~5 min
Anotar los triggers actuales de JR importante
En script.google.com con la cuenta jr@privegroup.com, ir a cada proyecto → Triggers (ícono de reloj). Screenshot o anotar qué función, qué intervalo. Esto es tu backup por si necesitás volver atrás.
Anotar los triggers actuales de Amanda
Mismo proceso en script.google.com con la cuenta amanda@privegroup.com.
Tener los archivos .gs listos para pegar
Abrí JR_Inbox_Manager.gs y Amanda_Agent_Core.gs del zip en un editor de texto. Los vas a copiar-pegar.
Tener a mano el Telegram bot token y chat ID
Lo necesitás para el paso de setTelegramToken() en Amanda. Si no lo tenés, buscá en el script v2.1-FINAL viejo (donde estaba hardcodeado).
Cuenta de JR (jr@privegroup.com)
1
Desactivar scripts viejos de JR
~5 min
Eliminar trigger del "Untitled project" crítico
Proyecto "Untitled project" → Triggers → Eliminar el trigger de jrTriageInbox_FULL (cada 5 min). Este es el que más quema API quota.
Eliminar trigger del "Labels gmail"
Proyecto "Labels gmail" → Triggers → Eliminar el trigger de jrTriageInbox_FULL (cada 15 min).
Eliminar trigger de "Amanda - Email Router (PRIVE GROUP)"
Proyecto "Amanda - Email Router" → Triggers → Eliminar el trigger de forwardToAmanda (cada 15 min).
Verificar que JR tiene 0 triggers activos
Ir a script.google.com → My Triggers (menú izquierdo). Debe mostrar 0 triggers activos para JR. Si ves alguno, eliminalo.
⚠ No borres los proyectos, solo los triggers. El código queda como backup.
2
Instalar script nuevo de JR
~5 min
Crear proyecto nuevo en Apps Script
En script.google.com (cuenta JR) → Nuevo proyecto → Renombrar a "JR Inbox Manager v3"
Pegar el código de JR_Inbox_Manager.gs
Borrar el contenido default del Code.gs → Pegar todo el contenido de JR_Inbox_Manager.gs → Guardar (Ctrl+S).
Test manual: ejecutar testRun() importante
Seleccionar la función testRun en el dropdown → Run. La primera vez pide permisos de Gmail — autorizá con la cuenta JR. Revisar el log (Ver → Logs) para confirmar que procesó sin errores.
⚠ Si da error de permisos, hacé click en "Advanced" → "Go to JR Inbox Manager v3 (unsafe)" → Allow.
Activar trigger: ejecutar setupTrigger() una vez
Seleccionar setupTrigger → Run. Esto crea el trigger automático de cada 30 minutos. Verificar en Triggers que aparece.
Cuenta de Amanda (amanda@privegroup.com)
3
Desactivar scripts viejos de Amanda
~5 min
Eliminar TODOS los triggers del "Agent Router v2.1-FINAL"
En script.google.com (cuenta Amanda) → Proyecto v2.1-FINAL → Triggers → Eliminar todos.
Eliminar TODOS los triggers del "Draft Queue Router (FULL)"
Proyecto Draft Queue Router → Triggers → Eliminar todos.
Verificar 0 triggers activos en la cuenta Amanda
My Triggers → debe estar vacío.
4
Instalar script nuevo de Amanda
~8 min
Crear proyecto nuevo en Apps Script
En script.google.com (cuenta Amanda) → Nuevo proyecto → Renombrar a "Amanda Agent Core v3"
Pegar el código de Amanda_Agent_Core.gs
Borrar Code.gs default → Pegar todo Amanda_Agent_Core.gs → Guardar.
Configurar credenciales: ejecutar setTelegramToken() crítico
Primero editá la función setTelegramToken() en el código — reemplazá YOUR_BOT_TOKEN_HERE y YOUR_CHAT_ID_HERE con los valores reales. Opcionalmente poné el Sheet ID para logging. Ejecutá la función. Después borrá los valores del código y guardá de vuelta.
⚠ Los tokens quedan guardados seguros en PropertiesService. Una vez ejecutado, borrá los valores hardcodeados del código.
Crear labels: ejecutar bootstrapLabels() una vez
Esto crea todos los labels AMANDA/QUEUE/*, AMANDA/STATE/*, etc. que el script necesita. Revisar log para confirmar.
Test manual: ejecutar routeAmandaQueue()
Autorizar permisos cuando pida. Verificar en logs que no hay errores. Si no hay emails en queue, va a decir "0 tasks processed" — está bien.
Activar triggers: ejecutar setupAmandaTriggers() una vez
Crea 3 triggers: routeAmandaQueue (15 min), monitorHITL (30 min), morningBrief (8 AM ET diario). Verificar en Triggers que aparecen los 3.
Validación
5
Verificar que todo funciona
~5 min
Test end-to-end: mandarte un email de prueba importante
Desde otra cuenta, mandá un email a jr@privegroup.com con subject tipo "Test opportunity deal acquisition" para activar la regla Opportunity RE. Esperá 30 min (o ejecutá testRun() manual en JR). Verificá que se labeleó y se forwardeó a Amanda.
Verificar que Amanda recibió y procesó
Esperá 15 min (o ejecutá routeAmandaQueue() manual en Amanda). Verificá que se creó un draft en Amanda con el payload AMANDA_TASK.
Verificar Telegram
Ejecutá morningBrief() manual en Amanda. Deberías recibir el brief en Telegram. Si no llega, revisá que el token y chat ID se guardaron bien (Settings → Script properties).
Limpieza (Opcional — hacer en 1 semana)
6
Limpiar scripts viejos
cuando estés seguro
Esperar 1 semana sin problemas
Dejá los proyectos viejos sin triggers por 1 semana. Si todo funciona bien con los nuevos, recién ahí limpialos.
Rotar el Telegram bot token
Pedí un nuevo token a @BotFather en Telegram. Actualizá en Amanda → Script properties. El token viejo (que estaba hardcodeado en v2.1-FINAL) queda invalidado.
