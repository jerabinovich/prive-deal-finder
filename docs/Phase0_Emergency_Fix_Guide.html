🔒 Security Fix: Move Telegram Token to PropertiesService

6 Move Hardcoded Telegram Bot Token to PropertiesService SECURITY

Why: Hardcoded secrets in source code are a security risk. PropertiesService encrypts and stores them securely.

⚠️ Dev Task: This step requires code editing. If you're not comfortable with code, ask your development team to complete this step.
Step 6a: In any script that uses a Telegram bot token, open the script editor (double-click project → Click Editor)
Step 6b: Find the line that looks like:
const TELEGRAM_BOT_TOKEN = "123456:ABCdef...xyz";
Step 6c: Replace that line with this code:
// Get token from secure storage const TELEGRAM_BOT_TOKEN = PropertiesService.getUserProperties().getProperty('TELEGRAM_BOT_TOKEN'); if (!TELEGRAM_BOT_TOKEN) { throw new Error('Telegram bot token not set. Please contact admin.'); }
Step 6d: To set the token initially, run this code once in the Console:
PropertiesService.getUserProperties().setProperty('TELEGRAM_BOT_TOKEN', 'YOUR_ACTUAL_TOKEN_HERE');
Step 6e: Click "Save" (Ctrl+S or Cmd+S)
The token is now stored securely in PropertiesService. The source code no longer contains the hardcoded secret.
